Identifying Errors in Jupyter Notebooks

Do you use Jupyter Notebooks and/or Python for your coding/scripting activities? If so, I'd like to learn more about how you work with the tool and identify errors within data analyses.


I would like to observe you as you go through a pre-written Jupyter Notebook and indentify errors which have been written into the notebook. Such an observation would take place over Zoom and you may participate from anywhere you feel comfortable. Participation will take roughly 1 hour of your time. We hope that from these observations we can make suggestions on how to more effectively teach error identification methods. The notebooks contain markdown, code, and data visualizations and each of these components may contain errors.


The errors within the notebooks fall within three categories: coding, data, and statistics. These categories are not mutally exclusive as an error may be some combination of the categories.


  • Coding: errors in which the code written doesn't properly achieve the task
  • Data: errors in which data cleaning or processing is done incorrectly
  • Statistics: errors in which incorrect statistical methods are used or incorrect statistical conclusions are drawn

I would also like to stress that we are not evaluating your skills but rather trying to learn from how you approach the task of identifying data analysis errors.
